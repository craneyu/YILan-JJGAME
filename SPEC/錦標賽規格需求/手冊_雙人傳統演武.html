§1系統概述

「宜蘭縣柔術錦標賽」是平台的第二個賽會類型，與既有「宜蘭縣運動會」共用同一套計分系統與認證架構，但賽制有顯著差異：增加二級分組（年齡/級別）、國小組規則簡化、不同年齡組獨立排名與頒獎。

1.1 兩種賽會類型對照

項目宜蘭縣運動會（sports-day）宜蘭縣柔術錦標賽（tournament）
分組僅一級（男/女/混合）一級 × 二級（年齡/級別）
排名群組依 category依 (category, tier)
賽程進行R1 全部隊伍跑 A → R2 跑 B → R3 跑 C同 tier 內 R1 → R2 → R3 跑完，再換 tier
VR 裁判所有組別國小組（EL/EM/EH）無 VR
創意演武時間罰則所有組別國小組免罰則
寢技預設時長3 分鐘依 tier：國小 2 分、青少年 3 分、公開 5 分
成績匯出依 category 一份檔案依 (category × tier) 拆分多份檔案
向後相容保證：啟用錦標賽功能對既有運動會行為零影響——所有運動會 events 的 meetingType 自動為 sports-day、tier 為 null，排序、計分、觀眾顯示、匯出全部與升級前完全一致。

§3競賽整體流程

賽程推進的核心原則：(tier, category) 群組順序完全依 Excel 匯入順序。每隊上台後依該 tier 規則跑完所有 round 才換下一隊。

群組順序規則：系統會將匯入的 Excel 表逐列掃描，遇到新的 (tier, category) 組合就建立一個新群組，相同組合會合併到既有群組。最終群組執行順序 = 該組合在 Excel 中首次出現的列順序。
不再有「組別順序」設定。承辦人員可以透過調整 Excel 的列順序直接控制整場賽程的進行順序。

圖 3-1：競賽整體流程。每隊依 tier 規則跑完所有 round/動作，再換下一隊；群組順序完全依 Excel 列順序。

3.1 三種 tier 流程模型

tier流程模型VR說明
EL EM 單輪連續演練 無 每隊上台後一次演完 A + B 系列所有動作，再換下一隊。無 C 系列、無 round 切換。
EH 三輪逐隊推進 無 同一隊伍依序跑 R1 (A) → R2 (B) → R3 (C) 完整 3 輪，再換下一隊。同 tier 內依 Excel 列順序推進。

3.2 自動跳過空群組

若 Excel 沒有列出某個 (tier, category) 組合，系統自動跳過——根本不會在群組序列中產生。例如本次 17 隊資料沒有「EL-female」「EM-male」等，整場流程從不會出現在這些群組。

3.3 單一隊伍組別處理

若某 (tier, category) 群組僅有一隊報名，該隊依該 tier 規則完成所有動作（含 VR，若適用）後，系統自動推進到下一群組。觀眾顯示自動隱藏 R/G 標籤，例如顯示 MALE EH 而非 MALE EH R1-G1。錦標賽中單隊組別相當常見（例：17 隊資料有 8 個單隊組別）。

3.4 群組順序由 Excel 直接決定

承辦人員只需要把 Excel 的隊伍列順序排好就完成「進場順序」的設定——系統不再提供「組別順序拖曳」UI（錦標賽不顯示，運動會仍保留）。

實務建議：建議匯入前先在 Excel 內以「年齡 → 性別」順序排好（例如 國小低年級男 → 國小低年級女 → 國小低年級混 → 國小中年級…），讓賽程符合常見的頒獎節奏。但若有特殊需求（例如外賓組要安排在中段），直接調 Excel 列順序即可。

§5計分演算法

5.1 每動作計分

每個動作（A1、B2、C3 等）由 5 位裁判獨立評分，每位裁判針對該動作的若干個評分項目（PART）給分。

評分項目（PART）數量

系列PART 數每 PART 最高分單動作最高分
A / B 系列4 個（P1～P4）3 分4 × 9 = 36 分
C 系列5 個（P1～P5）3 分5 × 9 = 45 分

「去頭尾取中間 3」演算法

每個 PART 由 5 位裁判給 0～3 分。系統處理流程：

  1. 收集 5 位裁判對某 PART 的分數，例如 [3, 3, 2, 2, 1]
  2. 排序後去掉最高 3 與最低 1，剩下中間 3 位 [2, 2, 3]
  3. 加總中間 3 位 = 該 PART 的得分（範例為 7 分，最高 9 分）
  4. 所有 PART 加總 = 該動作得分
計算範例：5 位裁判對 A1 動作給分如下：
裁判P1P2P3P4
13333
22332
33223
42232
51221
中間 3 加總7787

動作 A1 得分 = 7+7+8+7 = 29 分

5.2 系列總分與全場總分

系列總分 = 該系列所有動作得分加總 + VR 多樣性分數（若該 tier 適用）。

VR 多樣性評分

項目分數範圍說明
摔技多樣性0 / 1 / 2 分該系列展現的摔技種類豐富度
地板技多樣性0 / 1 / 2 分該系列展現的地板技種類豐富度
每系列 VR 最高4 分2 + 2 = 4 分
注意：國小組（EL/EM/EH）不適用 VR 評分。其系列總分 = 動作得分加總（不含 VR）。

5.3 排名

全場總分 = A 系列總分 + B 系列總分 + C 系列總分（若有）。同一 (category, tier) 群組內依全場總分由高到低排名，前 3 名頒獎。

§7操作指引

7.1 賽前準備

  1. 建立賽會：管理員登入 → 點「建立賽會」→ 填寫名稱、日期、場地 → 賽會類型選「宜蘭縣柔術錦標賽」 → 勾選運動項目（雙人演武等）→ 確認建立
  2. 下載匯入範本：進入該賽會 → 演武管理 → 點「範本」按鈕 → 下載含「分級」欄位與「分級對照表」的 Excel
  3. 填寫名單：將參賽隊伍填入範本，欄位包含「隊伍名稱、隊員一姓名、隊員二姓名、組別、分級」。分級欄位可填中文（如「青少年國中組」）或英文代碼（如「JH」）
  4. 上傳匯入：點「匯入雙人演武」→ 選擇剛填好的 Excel → 系統驗證後顯示「成功匯入 N 筆」
  5. 調整組別順序（可選）：管理介面可拖曳調整 male/female/mixed 進場順序
  6. 登入測試：在各裁判端登入並確認連線正常（OK 後再開賽）

7.2 賽中操作

  1. 賽序裁判選定第一隊（系統自動選 EL 或第一個非空群組的首隊）
  2. 點「開放動作評分」開放當前動作（如 A1）
  3. 5 位計分裁判各自評分後送出
  4. 系統自動計算動作分數並廣播至觀眾頁
  5. 賽序裁判點「開放下一動作」直到該系列完成
  6. 若為國高中以上 tier，VR 裁判送出該系列多樣性分數
  7. 賽序裁判點「換組」推進到下一隊 / 下一群組 / 下一輪
  8. 重複步驟 2-7 直到賽事結束
單隊組別小提醒：該隊跑完所有動作後，「換組」按鈕會自動 enable，按下即可推進到下一個非空 (tier, category) 群組——不需要透過「選擇隊伍」手動切換。

7.3 賽後成績匯出

  1. 管理員進入演武管理頁的「成績排名」區塊
  2. 每個 (category, tier) 顯示為一張獨立卡片，含名次與總分
  3. 點該卡片的「Excel」按鈕匯出該群組的詳細成績檔（檔名如 男子組_青少年國中組_成績明細.xlsx）
  4. 點「PDF」按鈕開啟列印視窗，輸出可供裁判長簽署的 A4 橫式 PDF
  5. 國小組（EL/EM/EH）的匯出檔自動省略 VR 欄位

§9常見問題 FAQ

如果某個 (tier, category) 群組只有一隊報名，會怎樣？
該隊伍會獨自完成所有動作評分。觀眾頁自動隱藏 R/G 標籤（顯示如 MALE EH），完成後系統自動推進到下一個非空群組。無需任何手動切換。
EL 隊伍跑完 A1 之後 B1 怎麼開放？
同一隊上台演完 A1 後，賽序裁判直接點「開放下一動作」即會看到 B1 為可選按鈕。EL/EM 採單輪連續演練，A 與 B 系列由同一隊伍一氣呵成完成，中間不換組。
為什麼 EL 隊伍的觀眾頁排名表只看到 A1 和 B1 欄位？
因為 EL（國小低年級）每系列僅 1 動作且無 C 系列。排名表動態依 tier 渲染欄位數：EL 顯示 A1/B1、EM 顯示 A1/A2/B1/B2、EH 顯示 A1-A3/B1-B3/C1-C3、JH/SH/OPEN 男子組顯示 A1-A4/B1-B4/C1-C4、女子/混合則 A1-A3/B1-B3/C1-C3。
「分級」欄位的 Excel 可以填什麼？
系統接受英文代碼（EL/EM/EH/JH/SH/OPEN/ELEM）或中文名稱（國小低年級、青少年國中組、公開組等）。下載「範本」按鈕的第二張工作表有完整對照表。
既有「宜蘭縣運動會」的資料會不會被影響？
不會。運動會 events 的 meetingType 自動為 sports-day、所有 teams 的 tier 為 null。排序退化為原本的 category-primary，所有計分、排名、觀眾顯示、匯出行為與升級前完全相同。
VR 裁判可以對國小組評分嗎？
不可以。錦標賽國小組（EL/EM/EH）不需要 VR 評分。VR 裁判進入這些隊伍的頁面會看到「此組別不需要 VR 評分」提示，所有評分按鈕與錯誤攻擊標記都停用。系統後端也會拒絕來自國小組的 VR 提交與錯誤攻擊請求。
寢技預設時間在哪裡設定？
系統依 Match.tier 自動套用：國小組（ELEM）= 2 分鐘、青少年國中（JH）= 3 分鐘、青少年高中（SH）= 3 分鐘、公開組（OPEN）= 5 分鐘。裁判可在賽前操作介面手動調整為其他值。
中途有隊伍棄權怎麼處理？
賽序裁判點「標記棄權」即可跳過該隊的 VR 等待條件。觀眾頁會顯示紅色「棄權」標籤。可隨時取消棄權回到正常流程。
如果同一動作不小心開放錯了，可以修改嗎？
5 位裁判一旦送出後分數不可修改。但若該動作整個被誤判為「錯誤攻擊」，VR 裁判可標記該動作為錯誤攻擊（國高中以上 tier 才有此功能），該動作分數會歸零並計入無效。
服務怎麼啟動？
使用 Docker Compose：docker compose up --build。或單獨啟動 MongoDB（docker start jju-mongo）+ 後端（cd backend && npm run dev）+ 前端（cd frontend && npm start）。
