## Context

裁判介面目前的計分按鈕結構（每側）：
- 得分：`2分` / `3分` / `4分`（無 +1、無扣分）
- 優勢：`優勢`（單一加）、無減
- 警告：`+警告` / `-警告`（分開但未整合）
- 無 STALLING 功能

需依寢技實際評判流程重新排列，加入中文提示與 STALLING，移除寢技不用的 1 分。

## Goals / Non-Goals

**Goals:**

- 每側計分區改為兩行：正向行與負向行
- 正向行：`+2` / `+3` / `+4` / `+A 優勢` / `+P 警告` / `STALLING`
- 負向行：`-2` / `-3` / `-4` / `-A 優勢` / `-P 警告`
- STALLING 按鈕在每側各自獨立（針對拖延方記錄）
- 新增 `addStalling(side)` 方法，STALLING 累計作為警告計入

**Non-Goals:**

- 後端資料模型不變（STALLING 映射為警告，不新增欄位）
- 不改動計時器區塊
- 不改動比賽清單視圖

## Decisions

### STALLING 映射為 Penalty

STALLING（拖延）在寢技計分中等同於追加一次警告。不需新增後端欄位，直接在前端呼叫 `addWarning(side)`，視覺上按鈕獨立標示 STALLING，語意清楚又不增加後端複雜度。

### 按鈕兩行排列（正向 / 負向）

兩行設計比單行更清楚區分「加分」與「扣分」，降低賽場緊張時誤觸風險。
正向行使用綠色調（`bg-green-600/70`），負向行使用紅色調（`bg-red-700/60`），STALLING 使用橘色（`bg-orange-600/70`）。

### 移除 1 分按鈕

寢技規則中技術分為 2、3、4 分，無 1 分選項，一律不顯示 +1 / -1 按鈕。

## Risks / Trade-offs

- [風險] 誤觸負向扣分：已有「取消上一筆」按鈕作為保護機制 → 不需額外確認 dialog
- [取捨] STALLING 不獨立追蹤：若未來需統計 STALLING 次數，須擴充後端，目前以簡單映射優先
