## Context

目前管理員的「清除成績」功能只提供一個按鈕，一次清除全部競賽類型（雙人演武 Duo + 創意演武 Show）的所有成績。當賽事同時包含兩種競賽項目時，若只需修正或重測其中一種，仍需清除全部資料，風險高且不靈活。

後端 `clearEventScores` 已依 `event.competitionTypes` 分岔處理雙人演武與創意演武的資料集合，具備良好的擴展基礎：只需傳入 `type` 參數即可選擇性清除其中一種。

## Goals / Non-Goals

**Goals:**

- 管理員可選擇「僅清除雙人演武」、「僅清除創意演武」或「清除全部」
- 清除前顯示確認對話框，說明將清除的範圍
- 後端 API 接受可選的 `type` 查詢參數（`Duo` | `Show`），無參數則清除全部
- 前端管理員介面的清除按鈕拆分為三個操作入口，入口根據賽事的 `competitionTypes` 動態顯示

**Non-Goals:**

- 不提供逐筆成績復原（Undo）功能
- 不修改 VRScore 獨立清除邏輯
- 不影響其他角色介面

## Decisions

### 後端 API：`type` 查詢參數篩選

**決策**：在 `DELETE /events/:id/scores` 加入可選的 `?type=Duo|Show` 查詢參數，無參數則清除全部。

**理由**：不需新增路由，向後相容（現有前端不帶 type 參數呼叫，行為不變）。後端已有依 `competitionTypes` 分岔的邏輯，只需在分岔前加一個 filter 條件即可。

**備選方案捨棄**：改成兩個獨立端點（`DELETE .../scores/duo` 和 `.../scores/show`）—— 更清晰但需修改路由定義與前端呼叫，增加的複雜度不成比例。

### 前端 UI：三個獨立按鈕

**決策**：在清除成績區塊顯示最多三個按鈕：「清除雙人演武成績」、「清除創意演武成績」、「清除全部成績」，依賽事的 `competitionTypes` 動態顯示（單一項目賽事只顯示該項目按鈕 + 全部按鈕）。

**理由**：三個按鈕各自意圖明確，不需下拉選單或 radio 選擇，減少操作步驟。危險程度最高的「全部清除」維持最顯眼位置，其餘分項清除採較低對比度，視覺上區分風險等級。

**備選方案捨棄**：單一按鈕 + 下拉選單 —— 可減少 UI 元素但需要額外互動步驟，且「全部清除」的危險性較難藉 UI 區分。

### 確認對話框訊息

**決策**：使用 SweetAlert2，對話框說明中明確列出將清除的競賽類型名稱（中文），例如：「此操作將清除【雙人演武】所有成績，無法還原，確定繼續？」。

## Risks / Trade-offs

- [Race condition] 清除期間若有計分裁判同時送出成績，部分資料可能未被清除 → 接受（LAN 環境，管理員操作前應告知裁判暫停）
- [GameState 未重置] 分項清除後 GameState 會重置至 idle，但若賽事進行中清除，賽序裁判端的本地 UI 狀態不會自動重置 → 裁判需手動重整頁面
